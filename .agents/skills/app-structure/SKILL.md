# 앱 구조 가이드

## 하위 앱 models.py 관리

앱 내 하위 앱으로 `models.py`를 별도로 작성하는 경우, 상위 앱의 `models.py`에서 하위 앱의 `models.py`를 import해야 한다. 이 때 `# noqa` 주석을 이용하여 import 오류를 무시하도록 한다.

## 앱별 역할

| 앱 | 역할 |
|---|---|
| `core` | 프로젝트에서 공통적으로 사용되는 항목들을 모아놓는다. INSTALLED_APPS에는 포함되지 않지만, 공통 인증/미들웨어/추상 모델/베이스 ViewSet 및 Mixin 등을 둔다. |
| `helper` | 특정한 분류에 속하지 않는 항목들을 모아놓는다. |
| `external` | 외부 서비스와 연동하는 항목들을 모아놓는다. |
| `board` | 고객에게 정보를 제공하거나, 게시판 기능을 제공하는 항목들을 모아놓는다. 이러한 기능을 개발할 경우 `board` 앱의 하위 앱으로 개발한다. |

## 템플릿 디렉토리

`templates` 디렉토리는 django 템플릿 파일을 모아놓는다. 템플릿 디렉토리 하위에 앱과 동일한 이름의 디렉토리를 만들어서 템플릿 파일을 관리한다.
