## 마이페이지 화면
mateya어플의 개발 명세서이다. 자세히 언급되지않은 부분은 질문하거나, 실무적으로 적절한 방안으로 처리 후 사용자에게 해당 사항을 보고한다.

## 공통지침 - 1
모든 API 요청은 아래 상태를 고려한다.

- Loading
- Success
- Validation Error
- Network Error
- Server Error(5xx)
에러 발생 시 토스트 또는 입력_에러 컴포넌트를 활용한다.

## 공통지침 - 2
- 기존 컴포넌트와 스타일 토큰을 먼저 찾아서 재사용하고,
없을 때만 새 컴포넌트를 만들어줘.
- 반응형은 iPhone 16 Pro 기준 모바일 우선으로 맞춰줘.
- 입력창에대한 에러가 발생했을 경우, "입력_에러" 컴포넌트를 활용하는 방안으로 한다.
  - 예시: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2400-2630&t=MxUj4cCr1Lu8U3oK-4
  - Error color와 Error icon을 활용한다.
- 버튼이 비활성화 되어있으면 "버튼_비활성화"컴포넌트를 활용하는 방안으로 한다.
- 입력 필드는 unfocused일 때는 외곽선이 "iconbackground"이고, focused이면 "black"색상이다.
- 상황별로 실무적으로 적절한 애니메이션, 화면전환을 넣고 사용자에게 해당 사항을 보고한다.


## I-1.마이페이지
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1421&t=sJAEnbv9HRapZtWp-11

### 프로필
- 사용자 정보를 조회하여 프로필 영역에 표시한다.
- 프로필 영역에는 아래 정보가 제공된다.
  - 프로필 사진
  - 이름
  - 30일 내 접속 여부
  - 대표 언어
  - 대표 나라
  - 거주지
- 대표 언어와 대표 나라는 사용자 프로필 정보를 기준으로 표시한다.
- 데이터 출처는 내부 DB이다.

### 지표
- 사용자의 활동 지표를 표시한다.
- 지표 영역에는 아래 정보가 제공된다.
  - 참가 및 생성 활동 수
  - 친구 수
  - 작성 리뷰 수
- MVP에서는 각 지표에 대한 자세히보기 기능을 제공하지 않는다.

### 뱃지
- 사용자의 활동 뱃지를 표시한다.
- MVP에서는 사용자가 특정 카테고리 활동을 1개 이상 체험한 경우, 해당 카테고리의 뱃지를 발급한다.
- 백엔드는 조건을 충족한 뱃지 이름과 뱃지 이미지를 내려준다.
- 카테고리별 뱃지명은 아래와 같다.
  - 전통문화: traditional!
  - 스포츠/액티비티: active person
  - 지역축제: festive!
  - 음식체험: food lover
  - 언어교환: language sharing
  - 관광/산책: tourist
  - 공예: craftsman
- 데이터 출처는 내부 정책이다.

### 활동 이력
- 사용자가 참여하거나 생성한 과거 활동 내역을 조회한다.
- 활동 이력은 최근순으로 정렬한다.
- 마이페이지에서는 최근 3개의 활동만 표시한다.
- 각 활동 이력에는 아래 정보가 제공된다.
  - 활동 카테고리
  - 대표사진
  - 활동명
  - 활동날짜
  - 활동 시간
  - 비용
  - 별점
  - 참여인원수
  - 최대인원수
- 데이터 출처는 내부 DB이다.


## I-2.다른사람페이지
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1423&t=sJAEnbv9HRapZtWp-11

### 기능
- 다른 사용자의 프로필 페이지를 조회한다.
- 기본 화면 구성은 "I-1.마이페이지"와 동일하다.
- 프로필 정보, 지표, 뱃지, 활동 이력 정보를 표시한다.

### 친구추가 기능
- 다른 사람 페이지에는 친구추가 버튼이 존재한다.
- 친구 상태가 아닌 사용자의 경우 친구추가가 가능하다.
- 친구추가가 완료되면 해당 사용자와의 1:1 채팅방이 채팅방 목록에 생성된다.

### 친구삭제 기능
- 이미 친구로 등록된 사용자의 경우 친구삭제가 가능하다.
- 친구삭제 이후에는 친구 상태가 해제된다.

### 채팅방 생성
- 친구추가 시 1:1 채팅방 목록에 해당 사용자와의 채팅방이 표시된다.
- MVP에서는 별도의 친구목록 기능을 제공하지 않는다.
- 친구 관계는 1:1 채팅방 목록을 통해 확인하는 방식으로 처리한다.

### 최근활동 목록
- 사용자의 최근활동 목록을 조회한다.
- 최근활동 목록에는 아래 정보가 제공된다.
  - 활동
  - 카테고리
  - 활동 이름
  - 날짜
  - 시간

### 통계
- 최근활동 보기 화면에서는 사용자의 활동 통계를 표시한다.
- 통계 영역에는 아래 정보가 제공된다.
  - 전체 활동 수
  - 개설 활동 수
  - 참여 활동 수
  - 작성 후기 수


## I-4.사업자 마이페이지
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1424&t=sJAEnbv9HRapZtWp-11

### 마이페이지
- 사업자 사용자의 마이페이지 정보를 조회한다.
- 사업자 프로필 영역에는 아래 정보가 제공된다.
  - 사진
  - 이름
  - 한줄소개
  - 장소

### 통계
- 사업자 마이페이지에서는 운영 관련 통계를 표시한다.
- 통계 영역에는 아래 정보가 제공된다.
  - 모집중인 체험 수
  - 누적 참가자 수
  - 평균 평점
  - 받은 후기 수

### 운영중인 체험목록
- 사업자가 운영중인 체험 목록을 조회한다.
- 운영중인 체험 목록에는 아래 정보가 제공된다.
  - 활동
  - 카테고리
  - 이름
  - 시간
  - 참여 인원
  - 비용
  - 사진
  - 리뷰 평점
- 리뷰 평점은 클래스인 경우에만 표시한다.

### 한줄소개 수정
- 사업자 사용자는 한줄소개를 수정할 수 있다.
- 한줄소개는 최대 500자까지 입력할 수 있다.
- 저장 버튼 클릭 시 수정된 한줄소개가 반영된다.

## I-6.회원 탈퇴
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2404-1498&t=sJAEnbv9HRapZtWp-11

### 회원탈퇴팝업
- 회원탈퇴하기 버튼을 누르면 회원탈퇴팝업을 표시한다.
- 사용자는 팝업에서 회원탈퇴를 진행할 수 있다.
- 회원탈퇴 처리 시 계정은 즉시 hard delete하지 않고 soft delete 처리한다.
- soft delete 이후 30일이 지나면 최종 삭제한다.
- 회원탈퇴 진행 전, 사용자는 회원탈퇴와 관련한 개인정보 관리 방침에 동의해야 한다.
- 동의 절차는 서명 입력 방식으로 처리한다.

### 정책
- 탈퇴 요청 직후에는 계정을 비활성화 상태로 처리한다.
- 30일 이내 복구 정책이 필요할 경우, 복구 가능 여부와 복구 진입점을 별도로 정의해야 한다.
- 30일 이후 최종 삭제 시점에는 사용자 개인정보, 프로필, 인증 정보, 활동 데이터의 삭제 또는 익명화 범위를 백엔드 정책에 맞게 처리한다.
