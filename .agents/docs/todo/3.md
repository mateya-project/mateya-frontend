Flutter 앱에서 위치정보, 알림, 카메라, 사진, 마이크, 연락처 등 사용자 권한을 요청하는 모든 지점을 점검하고, App Store / Google Play 심사 기준에 맞게 수정해주세요.

## 핵심 원칙

- 앱 실행 직후 무조건 권한 팝업을 띄우지 마세요.
- 사용자가 해당 기능을 사용하려는 시점에 권한을 요청하세요.
- OS 권한 팝업 전에 앱 자체 안내 팝업을 먼저 보여주세요.
- 안내 문구에는 “왜 필요한지”, “어디에 쓰이는지”, “거부해도 어떤 기능은 사용 가능한지”를 명확히 적어주세요.
- 권한 요청은 실제 기능에 필요한 최소 범위만 사용하세요.
- 백그라운드 위치 권한은 핵심 기능에 반드시 필요한 경우가 아니면 요청하지 마세요.

Apple은 권한 요청 시 사용 목적을 명확하고 구체적으로 설명해야 한다고 안내하고, Google Play도 위치 권한이 핵심 기능에 필요한지와 사용자에게 명확한 고지가 있는지를 심사합니다. ## 작업 지침

1. 프로젝트 전체에서 권한 요청 코드를 검색해주세요.
    - location
    - permission_handler
    - geolocator
    - camera
    - photo
    - notification
    - microphone
    - contacts
    - bluetooth

2. 각 권한마다 다음을 정리해주세요.
    - 권한 종류
    - 요청 위치
    - 요청 타이밍
    - 실제 기능상 필요한 이유
    - 과도한 권한 여부
    - iOS / Android 설정 파일 반영 여부

3. OS 권한 팝업 전에 자체 안내 팝업을 추가해주세요.

예시 문구:

위치 권한:
“내 주변 서비스/매칭 결과를 보여드리기 위해 현재 위치 권한이 필요합니다. 위치 정보는 해당 기능 제공 목적으로만 사용됩니다.”

알림 권한:
“채팅 메시지, 예약 상태, 중요한 서비스 알림을 받기 위해 알림 권한이 필요합니다.”

사진 권한:
“프로필 이미지 또는 게시물 사진을 업로드하기 위해 사진 접근 권한이 필요합니다.”

카메라 권한:
“프로필 사진 촬영 또는 인증 사진 촬영을 위해 카메라 권한이 필요합니다.”

4. 권한 거부 처리도 구현해주세요.
    - 최초 거부: 기능 제한 안내 + 다시 요청 가능
    - 영구 거부: 설정 화면으로 이동 안내
    - 권한이 없어도 앱 전체가 멈추지 않도록 처리

5. iOS 설정을 점검해주세요.
    - Info.plist에 권한 설명 문구 추가
    - NSLocationWhenInUseUsageDescription
    - NSCameraUsageDescription
    - NSPhotoLibraryUsageDescription
    - NSMicrophoneUsageDescription 등
    - 문구는 “앱 기능과 직접 연결된 구체적인 설명”으로 작성

6. Android 설정을 점검해주세요.
    - AndroidManifest.xml 권한 선언 확인
    - ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION 구분
    - ACCESS_BACKGROUND_LOCATION은 꼭 필요한 경우가 아니면 제거
    - Android 13 이상 알림 권한 POST_NOTIFICATIONS 처리
    - Google Play 심사를 위해 위치 권한 사용 목적이 앱 기능과 일치하는지 확인

## 금지사항

- 앱 시작하자마자 권한 팝업 띄우기 금지
- “서비스 이용을 위해 필요합니다”처럼 모호한 문구 금지
- 실제로 쓰지 않는 권한 선언 금지
- 위치 권한을 광고, 분석, 추적 목적으로 요청 금지
- 백그라운드 위치 권한을 습관적으로 추가 금지
- 권한 거부 시 앱을 강제 종료하거나 진행 불가 상태로 만들지 말 것

## 최종 목표

사용자가 권한 요청 이유를 명확히 이해하고 동의할 수 있도록 하며, App Store / Google Play 심사에서 “불명확한 권한 요청”, “과도한 권한 요청”, “위치 권한 고지 부족”으로 반려되지 않도록 수정해주세요.
