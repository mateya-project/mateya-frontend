Flutter 앱 전체를 검토하여 실무적으로 적절한 위치에 애니메이션 효과를 추가해주세요.

단, 과도한 장식용 애니메이션이 아니라 UX를 자연스럽게 개선하는 목적이어야 합니다.

## 애니메이션 적용 기준

다음 상황에 우선적으로 적용해주세요.

1. 화면 전환
- 페이지 진입/뒤로가기 시 자연스러운 transition 적용
- 갑작스러운 화면 변경이 느껴지지 않도록 처리

2. 버튼/탭/카드 클릭
- 사용자가 눌렀다는 피드백이 느껴지도록 scale, opacity, ripple 등을 적용
- 중요한 CTA 버튼은 눌림 효과를 명확히 표시

3. 리스트/카드 표시
- 목록 데이터가 처음 로드될 때 fade-in 또는 slide-in 적용
- 단, 항목이 많은 리스트에서는 성능 저하가 없도록 과도한 stagger animation 금지

4. 로딩/빈 상태/에러 상태
- 로딩 중에는 skeleton, shimmer, progress animation 등을 적절히 사용
- 빈 화면은 일러스트/문구가 부드럽게 나타나도록 처리
- 에러 메시지는 갑작스럽게 뜨지 않도록 fade/slide 처리

5. 상태 변경
- 좋아요, 북마크, 체크박스, 토글, 신고하기 등 상태가 바뀌는 UI에 짧은 micro-interaction 적용
- 상태 변경 결과가 사용자에게 즉시 인지되도록 처리

6. Bottom Sheet / Dialog / Snackbar
- 등장/사라짐 애니메이션을 자연스럽게 적용
- 배경 dim, slide-up, fade 효과를 플랫폼 관례에 맞게 사용

## 구현 원칙

- 기본적으로 Flutter 내장 애니메이션 위젯을 우선 사용해주세요.
    - AnimatedContainer
    - AnimatedOpacity
    - AnimatedSwitcher
    - AnimatedScale
    - AnimatedSlide
    - Hero
    - PageRouteBuilder
- 복잡한 애니메이션이 필요한 경우에만 animation_controller를 사용해주세요.
- duration은 보통 150~300ms 범위로 유지해주세요.
- easing은 Curves.easeOut, Curves.easeInOut, Curves.fastOutSlowIn 등을 상황에 맞게 사용해주세요.
- 사용자의 행동을 방해하거나 앱이 느리게 느껴지는 애니메이션은 넣지 마세요.
- 스크롤 성능에 영향을 주는 애니메이션은 최소화해주세요.
- 같은 패턴의 애니메이션은 공통 위젯이나 유틸로 분리해주세요.
- 접근성을 고려하여 Reduce Motion 설정이 가능하면 애니메이션을 줄이거나 비활성화할 수 있게 해주세요.

## 작업 방식

코드를 수정하기 전에 먼저 다음을 작성해주세요.

1. 현재 화면별 애니메이션 적용 후보
2. 각 애니메이션의 목적
3. 적용할 Flutter 위젯 또는 패키지
4. 예상되는 UX 개선 효과
5. 성능상 주의할 부분

그 후 승인받은 뒤 실제 코드를 수정해주세요.

## 금지사항

- 모든 요소에 무분별하게 애니메이션 넣기 금지
- 500ms 이상 느린 애니메이션 남발 금지
- 리스트의 모든 아이템에 무거운 애니메이션 반복 금지
- 단순 장식 목적의 과한 bounce, rotate, glow 효과 금지
- 앱 전체 톤과 맞지 않는 화려한 효과 금지
- 비즈니스 로직과 애니메이션 코드를 과하게 섞지 말 것

목표는 “화려한 앱”이 아니라, 사용자가 조작 결과를 자연스럽게 이해할 수 있는 부드러운 앱으로 만드는 것입니다.
