Flutter 앱 헤더의 globe icon을 눌렀을 때 언어 변경 팝업이 뜨고, 실제로 앱 내 UI/UX 문구와 문서가 다국어로 변경되도록 구현해주세요.

지원 언어

다음 4개 언어를 지원해주세요.

* 한국어
* 영어
* 일본어
* 중국어

구현 목표

단순히 언어 선택 팝업만 보여주는 것이 아니라, 사용자가 언어를 선택하면 앱 내 표시 문구가 실제로 해당 언어로 변경되어야 합니다.

적용 범위

다음 영역의 텍스트를 모두 다국어 처리해주세요.

1. 화면 내 UI 문구

* 버튼
* 탭
* 헤더
* 안내 문구
* 빈 상태 문구
* 에러 메시지
* 성공 메시지
* 로딩 문구
* 권한 요청 안내 문구

2. 앱 내 문서

* 서비스 이용약관
* 개인정보 처리방침
* 개인정보 제3자 제공 동의
* 위치기반 서비스 이용약관
* 만 14세 이상 확인 문구
* 신고/차단 관련 안내 문구
* 고객센터/FAQ 문구

3. 팝업/바텀시트

* 언어 변경 팝업
* 권한 요청 팝업
* 신고하기 팝업
* 로그아웃/탈퇴 확인 팝업
* 오류/알림 Snackbar

언어 변경 UX

* 헤더의 globe icon 클릭 시 언어 선택 팝업 또는 bottom sheet을 띄워주세요.
* 현재 선택된 언어는 체크 표시로 구분해주세요.
* 언어 목록은 다음처럼 표시해주세요.
    * 한국어
    * English
    * 日本語
    * 中文
* 사용자가 언어를 선택하면 즉시 앱 문구가 변경되도록 해주세요.
* 선택한 언어는 앱 재실행 후에도 유지되도록 로컬 저장소에 저장해주세요.
* 언어 변경 후 앱을 강제 재시작해야 하는 구조는 피해주세요.

구현 방식

* Flutter 공식 localization 구조를 우선 사용해주세요.
* flutter_localizations, intl, ARB 파일 기반 구조를 권장합니다.
* 언어별 번역 파일을 분리해주세요.

예시 구조:

lib/
l10n/
app_ko.arb
app_en.arb
app_ja.arb
app_zh.arb

* 하드코딩된 텍스트는 모두 localization key로 분리해주세요.
* 동일한 문구는 중복 key를 만들지 말고 재사용해주세요.
* 날짜, 시간, 숫자, 통화 포맷도 언어 설정에 맞게 표시되도록 고려해주세요.

문서 번역 기준

* 약관/개인정보 문서는 단순 자동번역 느낌이 아니라 앱 심사와 실사용에 적합한 자연스러운 문장으로 작성해주세요.
* 법적 의미가 달라지지 않도록 원문 구조를 유지해주세요.
* 각 언어별 문서 내용은 별도 파일 또는 localization resource로 관리해주세요.
* 약관 상세보기 화면에서도 현재 선택한 언어에 맞는 문서가 표시되어야 합니다.

품질 기준

* 한국어, 영어, 일본어, 중국어 중 하나라도 누락된 key가 있으면 안 됩니다.
* 지원하지 않는 언어 환경에서는 기본값을 한국어 또는 영어로 fallback 처리해주세요.
* 앱 내 일부 화면만 번역되고 일부는 원문으로 남는 상태를 만들지 마세요.
* 새로 추가되는 화면도 localization key를 사용하도록 구조화해주세요.
* 언어 변경 후 navigation, dialog, snackbar, form validation 메시지도 함께 변경되어야 합니다.

작업 순서

코드를 수정하기 전에 먼저 다음을 작성해주세요.

1. 현재 하드코딩된 텍스트 목록
2. 다국어 처리 대상 화면/문서 목록
3. 생성할 localization 파일 구조
4. 언어 변경 상태 관리 방식
5. 선택 언어 저장 방식
6. fallback 언어 정책

그 후 Flutter localization best practice 기준으로 구현해주세요.

금지사항

* globe icon 클릭 시 UI만 뜨고 실제 언어 변경은 안 되는 구현 금지
* 일부 문구만 번역하고 나머지는 하드코딩으로 남기는 것 금지
* 약관/개인정보 문서를 한 언어로만 제공하는 것 금지
* 번역 문구를 위젯 내부에 직접 하드코딩하는 것 금지
* 언어 변경을 위해 앱 재설치를 요구하는 구조 금지

최종 목표는 사용자가 globe icon에서 언어를 변경하면 앱 전체 UI와 주요 문서가 한국어/영어/일본어/중국어로 자연스럽게 전환되는 다국어 앱을 만드는 것입니다.
