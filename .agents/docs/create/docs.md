# MateYa 생성/등록 화면 기능 명세서

mateya어플의 생성/등록 화면 개발 명세서이다. 자세히 언급되지 않은 부분은 질문하거나, 실무적으로 적절한 방안으로 처리 후 사용자에게 해당 사항을 보고한다.

## 공통지침 - 1
모든 API 요청은 아래 상태를 고려한다.

- Loading
- Success
- Validation Error
- Network Error
- Server Error(5xx)

에러 발생 시 토스트 또는 입력_에러 컴포넌트를 활용한다.

## 공통지침 - 2
- 기존 컴포넌트와 스타일 토큰을 먼저 찾아서 재사용하고, 없을 때만 새 컴포넌트를 만들어줘.
- 반응형은 iPhone 16 Pro 기준 모바일 우선으로 맞춰줘.
- 입력창에 대한 에러가 발생했을 경우, "입력_에러" 컴포넌트를 활용하는 방안으로 한다.
- 버튼이 비활성화 되어있으면 "버튼_비활성화" 컴포넌트를 활용하는 방안으로 한다.
- 입력 필드는 unfocused일 때는 외곽선이 "iconbackground"이고, focused이면 "black" 색상이다.
- 상황별로 실무적으로 적절한 애니메이션, 화면전환을 넣고 사용자에게 해당 사항을 보고한다.

## C-1.모임 생성 화면
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1412&t=sJAEnbv9HRapZtWp-11

### 기능
- 모임 생성의 첫 단계 화면이다.
- 유저는 생성할 모임의 유형을 선택한다.
- 현재 선택 가능한 유형은 "한국문화 체험"이다.
- 한국문화 체험 유형을 선택한 뒤, 카테고리를 지정한다.
- 카테고리는 필수 선택값이다.

### 카테고리 목록
- 전통문화
- 스포츠/액티비티
- 지역축제
- 음식체험
- 언어교환
- 관광/산책
- 공예
- 기타

### 정책
- 카테고리는 1개 이상 반드시 선택되어야 한다.
- 카테고리 미선택 상태에서는 다음 단계 이동 버튼을 비활성화한다.
- 비활성화 버튼은 "버튼_비활성화" 컴포넌트를 활용한다.
- 선택값은 다음 단계인 "C-2.모임 생성(2단계) 화면"으로 전달한다.

## C-2.모임 생성(2단계) 화면
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1409&t=sJAEnbv9HRapZtWp-11

### 기능
- 모임 생성의 두 번째 단계 화면이다.
- 유저는 모임이 진행될 장소를 검색하고 선택한다.
- 장소 검색은 키워드 기반으로 동작한다.
- 검색 대상은 공공시설 및 카테고리별 공공데이터를 우선 활용한다.
- 검색 결과 중 하나를 선택하면 지도 영역에 해당 위치를 표시한다.
- 지도 표시는 프론트에서 처리한다.
- 지도 API는 카카오맵 API 또는 네이버맵 API를 활용한다.

### 장소 검색
- 장소명을 텍스트로 입력하여 검색한다.
- 검색 결과는 장소 이름 매칭을 기준으로 제공한다.
- 유저는 검색 결과 중 1개를 선택할 수 있다.
- 선택된 장소는 다음 단계인 "C-3.모임 생성(3단계) 화면"으로 전달한다.

### 장소 추천
- 유저가 설정한 동네 위치를 기준으로 동일 카테고리 내 장소를 추천한다.
- 추천 장소는 가까운 순으로 최대 3개 제공한다.
- 추천 목록은 공공데이터 기반으로 구성한다.

### 카테고리별 데이터 활용
- 스포츠/액티비티: 한국관광공사 데이터
- 지역축제: 지역축제 공공데이터
- 전통문화: 전통문화유산 공공데이터
- 관광/산책: 관광지 관련 데이터 활용 예정

### 지도 조회
- 검색 결과 또는 추천 목록에서 선택한 장소의 위치를 지도에 표시한다.
- 지도 이동, 확대/축소 등 기본 지도 인터랙션은 지원한다.
- 위치 데이터가 없는 경우 입력_에러 컴포넌트 또는 토스트로 안내한다.

### 정책
- 장소는 1개 필수 선택이다.
- 장소 미선택 상태에서는 다음 단계 이동 버튼을 비활성화한다.
- 검색 결과가 없을 경우 빈 상태 화면을 제공한다.
- 네트워크 오류 또는 지도 API 오류 발생 시 토스트로 안내한다.

## C-3.모임 생성(3단계) 화면
- url: https://www.figma.com/design/BqIBscM2a1oAFrIuF8lqvK/Potfolio-Template--%EC%9D%B4%EB%A0%A5%EC%84%9C-%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%ED%85%9C%ED%94%8C%EB%A6%BF---Community-?node-id=2339-1410&t=sJAEnbv9HRapZtWp-11

### 기능
- 모임 생성의 세 번째 단계 화면이다.
- 유저는 모임 제목, 소개, 일정, 시간, 모집 인원, 모집 마감일, 진행 언어, 비용, 참가 대상, 대표 이미지를 입력한다.
- 필수 입력값을 모두 만족하면 모임 생성이 가능하다.
- 수정하기 화면으로 들어온 경우에는 모임 삭제 기능을 제공한다.
- 모임 생성 완료 시 해당 모임에 대한 채팅방이 자동 생성된다.

### 제목 입력
- 모임명을 작성한다.
- 최대 100자까지 입력 가능하다.
- 필수 입력값이다.
- 미입력 또는 글자 수 초과 시 입력_에러 컴포넌트를 활용한다.

### 소개 입력
- 모임 설명을 작성한다.
- 최대 3000자까지 입력 가능하다.
- 선택 입력값이다.
- 글자 수 초과 시 입력_에러 컴포넌트를 활용한다.

### 날짜 설정
- 모임 날짜를 선택한다.
- DatePicker를 사용한다.
- 과거 날짜는 선택할 수 없다.
- 필수 입력값이다.

### 시작시간 설정
- 모임 시작 시간을 설정한다.
- 필수 입력값이다.

### 종료시간 설정
- 모임 종료 시간을 설정한다.
- 필수 입력값이다.
- 종료 시간은 시작 시간보다 늦어야 한다.
- 잘못된 시간 범위가 입력될 경우 입력_에러 컴포넌트를 활용한다.

### 인원 설정
- 모집 인원을 설정한다.
- 최소 2명, 최대 20명까지 지정할 수 있다.
- 필수 입력값이다.

### 모집 마감 날짜 설정
- 모집 마감 날짜를 설정한다.
- 모임 날짜 이후의 날짜는 선택할 수 없다.
- 과거 날짜 선택은 불가하다.

### 모집 마감 시간 설정
- 모집 마감 시간을 설정한다.
- 모집 마감 일시가 현재 시각보다 이전이면 입력_에러 컴포넌트를 활용한다.
- 모집 마감 일시는 모임 시작 일시보다 이전이어야 한다.

### 언어 설정
- 모임 진행 언어를 선택한다.
- 다중 선택이 가능하다.
- 1개 이상 필수 선택이다.

### 비용
- 무료 또는 유료 중 하나를 선택한다.
- 필수 입력값이다.
- 무료를 선택한 경우 가격 입력 필드는 비활성화하거나 숨긴다.
- 유료를 선택한 경우 가격을 입력한다.
- 유료 가격은 0원 이상 200,000원 이하로 제한한다.

### 참가대상
- 참가자 조건을 설정한다.
- 다중 선택이 가능하다.
- 선택 가능한 항목은 아래와 같다.
  - 누구나
  - 외국인 환영
  - 한국인 환영
  - 관광객 추천
  - 초보자 환영

### 이미지 등록
- 모임 대표 이미지를 업로드한다.
- 최대 5장까지 등록할 수 있다.
- 대표사진을 설정할 수 있다.
- 이미지 1장당 최대 용량은 10MB 이하로 제한한다.
- 이미지 저장소는 S3를 활용한다.
- 이미지 형식, 용량, 개수 제한을 초과하면 입력_에러 컴포넌트 또는 토스트로 안내한다.

### 모임 삭제하기
- 수정하기로 들어온 경우에만 모임 삭제 버튼을 노출한다.
- 삭제 버튼을 누르면 확인 팝업을 띄운다.
- 삭제 완료 후 이전 화면 또는 내 모임 관리 화면으로 이동한다.

### 채팅방 생성
- 모임 생성과 동시에 해당 모임 전용 채팅방을 자동 생성한다.
- 채팅방 생성 실패 시 모임 생성 성공 여부와 채팅방 생성 실패 상태를 분리하여 처리한다.
- 실패 시 유저에게 토스트로 안내하고, 재시도 가능한 정책을 둔다.

### 필수 입력값
- 제목
- 날짜
- 시작시간
- 종료시간
- 모집 인원
- 진행 언어 1개 이상
- 비용 유형
- 유료 선택 시 가격
- 대표 이미지

### 정책
- 필수 입력값이 모두 충족되기 전까지 생성 버튼은 비활성화한다.
- 생성 버튼 비활성화 상태에서는 "버튼_비활성화" 컴포넌트를 활용한다.
- 모든 입력값은 프론트 1차 검증과 서버 2차 검증을 모두 수행한다.
- 생성 성공 시 상세보기 화면 또는 생성 완료 화면으로 이동한다.
- 생성 실패 시 오류 유형에 맞게 토스트 또는 입력_에러 컴포넌트를 활용한다.

- 삭제 완료 후 이전 화면 또는 내 클래스 관리 화면으로 이동한다.

### 채팅방 생성
- 클래스 생성과 동시에 해당 클래스 전용 채팅방이 자동 생성된다.
- 채팅방 생성 실패 시 클래스 생성 성공 여부와 채팅방 생성 실패 상태를 분리하여 처리한다.
- 실패 시 유저에게 토스트로 안내하고, 재시도 가능한 정책을 둔다.

### 필수 입력값
- 클래스 이름
- 날짜
- 시작시간
- 종료시간
- 모집 인원
- 진행 언어 1개 이상
- 비용 유형
- 유료 선택 시 가격
- 장소

### 정책
- 필수 입력값이 모두 충족되기 전까지 등록 버튼은 비활성화한다.
- 등록 버튼 비활성화 상태에서는 "버튼_비활성화" 컴포넌트를 활용한다.
- 모든 입력값은 프론트 1차 검증과 서버 2차 검증을 모두 수행한다.
- 등록 성공 시 상세보기 화면 또는 등록 완료 화면으로 이동한다.
- 등록 실패 시 오류 유형에 맞게 토스트 또는 입력_에러 컴포넌트를 활용한다.
